Web foundations

HTML · intent encoded in markup

Markup is choreography: every heading level commits to an outline, every landmark guides assistive journeys, every control answers “what happens when invoked?” HTML is rarely glamorous—yet flagship experiences still rest on ruthless semantics.

Semantic evolution

From Berners-Lee’s physics hypertext spine to WHATWG’s living HTML, the language absorbed forms, realtime media, dialog semantics, intrinsic lazy loading—without abandoning graceful degradation older clients expect.

Elements such as article, section, nav, aside, header, footer, and main let machines infer region intent; pairing them thoughtfully beats endless <div> archaeology.

Pattern · document scaffold

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>Module · concise intent</title>
</head>
<body>
  <a class="skip" href="#primary">Skip to lesson</a>
  <header>…navigation…</header>
  <main id="primary">…</main>
  <footer>…</footer>
</body>
</html>

Accessibility & SEO as shared optics

Assistive stacks and crawler heuristics both honor heading order, descriptive link phrases, captions tied to multimedia, labeling that survives CSS failure. Baking those checks inside CI—not “later QA”—is how instructional sites preserve trust.

Alliance with CSS & JavaScript

  • HTML models capabilities (dialog open states, checkbox indeterminates, muted tracks).
  • CSS narrates temperament—spacing rhythm, kinetic cues honoring prefers-reduced-motion.
  • JavaScript augments—but only after native affordances show their seams; redundant ARIA is a scent of missing semantics.
Ship prototypes in three modalities: keyboard-only traversal, synthesized speech, images disabled. If narration stumbles, refactor markup before palettes.

Habits worth installing

Validators and axe-style audits catch heading skips early; forbid placeholder-only labeling; annotate live regions sparingly yet intentionally for dynamic quizzes.

Forms: where user intent becomes bytes

Inputs must carry deterministic names into server or client routers; radios/checkboxes expose value sets; selects group options logically. Prefer explicit label for=... binding over orphaned placeholders—assistive stacks announce labels even when placeholders vanish mid-typing.

  • method="GET": idempotent reads; payloads appear in URLs and logs—fine for searches, careless for passwords.
  • method="POST": bodies carry heavier state; pair with CSRF defenses when authenticated.
  • Client validation is UX sugar; authoritative validation happens server-side (or mirrored in trusted WASM)—never trust the browser alone for integrity.

Metadata learners actually notice

<title> and descriptive meta name="description" power search snippets; social cards lean on OpenGraph/Twitter equivalents when you syndicate coursework pages. Charset and viewport declarations belong first in <head> to avoid ambiguous parsing flashes on slow networks.

    Annotate GATE-style memory hierarchies alt-text-first; sighted learners get captions with exam vocabulary.

Security-flavored primitives

Outbound links targeting new browsing contexts merit rel="noopener noreferrer"; user-generated snippets demand escaping—prefer framework defaults over hand-rolled substitutions. Sensitive flows avoid embedding secrets in markup (attributes leak to extensions and logs).

Tables aren’t villains—misuse is

When tabular comparisons (topic coverage, GATE weighting) outperform card stacks, revive <table> with <th scope>, captions, and responsive patterns (horizontal scroll wrappers) instead of inaccessible div grids masquerading as spreadsheets.

Teach every template as two documents: one read aloud via screen reader, one visually styled. Divergence signals you’ve smuggled meaning into CSS-only channels.
