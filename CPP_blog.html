Systems-oriented track

C++ · where abstractions negotiate with silicon

Born as “C with classes,” ISO C++ now balances zero-cost abstraction, deterministic resource control, generic programming, and a standard library steeped in iterators and value semantics—a toolkit for infra, embedded, games, compilers, browsers.

Historical anchor

Bjarne Stroustrup extended C at Bell Labs toward safer large-scale modeling. Today’s lineage (C++11 through C++23 and beyond) front-loads RAII, smart pointers, concurrency primitives, constexpr evaluation, ranges, modules—yet unforgiving Undefined Behavior still waits for careless pointer games.

Pillars practitioners lean on

  • Classes & encapsulation: invariants guarded by constructors—not “OOP trivia,” but correctness contracts.
  • Composition & reuse: templates + concepts express algorithms decoupled from concrete containers.
  • Deterministic lifetimes: destructors free locks, sockets, descriptors—paired with exceptions or noexcept policies.
  • Value semantics: cheap moves, disciplined copies—STL containers assume this etiquette.

Why enterprises still fund C++

  • Predictable performance envelopes when profiles demand it.
  • Portability across OS + bare-metal corridors.
  • Interoperability with C ABIs powering OS kernels, interpreters, cryptographic libraries.

Sharp edges taught with honesty

  • UB on overflow, dangling references, uninitialized reads—Clang sanitizers deserve weekly runs.
  • Build times inflate without modules discipline and forward declarations hygiene.
  • Templates error messages escalate—readable boundaries via concepts and restraint.

Memory tiers in one breath

Automatic locals and call frames live alongside the stack—fast, scoped lifetimes; dynamic lifetimes gravitate toward the heap via new/malloc kin (prefer smart pointers instead of raw juggling). BSS/data hold globals and static-storage duration objects; literal strings may live in read-only mappings. Connecting this map with virtual memory lectures from COA anchors why prefetching and false sharing sting when code “looks harmless.”

STL map for nervous beginners

  • std::vector: contiguous amortized push; beware iterator invalidation after growth.
  • std::string_view: non-owning observers—lifetime discipline is on you.
  • std::unordered_map: average fast lookups; hashing quality matters adversarially but rarely pedagogically.
  • std::optional / std::variant: model partial results and algebraic shapes without sentinel magic numbers.

References versus pointers—and when each lies

References disguise addresses with syntax sugar (. chaining, no reassignment semantics); pointers expose arithmetic and reassignment—which is expressive for traversing graphs but hazardous without invariants. In modern codebases, pointers appear at ABI boundaries while references carry internal contracts—as long as they never alias expired storage.

Const correctness is documentation that compiles

#include <vector>

double mean(const std::vector<int>& nums) noexcept {
    if (nums.empty()) return 0.0;
    long long acc = 0;
    for (int x : nums) acc += x;
    return static_cast<double>(acc) / nums.size();
}

Mark immutable parameters const& to widen sharing without copies; annotate methods const when they uphold logical immutability so accidental mutation trips the compiler—not your future self at 02:00 before mock GATE.

Move semantics in a paragraph

Sometimes cheap “copy” constructs by stealing resources (heap buffers, file handles): rvalue references flag expiring temporaries enabling moves. Respect the Rule of Five when you customize destruction—exceptions, moves, swaps, copy—because half-implemented RAII shells rot quietly until sanitizers shout.

Weekly drill: pick one Clang/GCC diagnostic you fear (template instantiation cliffs, narrowing conversions) and solve it bare-handed before hiding behind macros—pain now pays in OS and compiler GATE stems.
